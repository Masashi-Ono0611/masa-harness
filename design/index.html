知能（skill）・実行（hook）・記憶（memory）を別レイヤに分離した、自分専用の operating system。

01  設計哲学

賢さは skill へ。執行は tool へ。

CLAUDE.md は pointer ＋ 必ず効く reflex だけに保ち、手順は書かない。知能・実行・記憶を分離する。

harness は配線図だけ持ち、賢さ・執行・記憶は別レイヤに分離する
PRINCIPLE 1
Think Before Coding
不明点は止めて質問。docs を grep してから訊く。
PRINCIPLE 2
Simplicity First
依頼を解く最小コードのみ。投機的な抽象化はしない。
PRINCIPLE 3
Surgical Changes
触るのは必要箇所のみ。概念を変えたら旧名で全域 grep。
PRINCIPLE 4
Goal-Driven Execution
検証可能な成功基準に変換。テストを通して報告。
02  全体構成

5 レイヤで分業する

入力は必ず ① の hook 層を通過してから実行。各層は config-hygiene の owner マップで「足す / 削る / 監査する」係が単一に決まっている（MECE）。

各層の owner が固定され、監査の重複（nag）も抜け（GAP）も起きない
① hooksconfig-audit
構造・permission・hook 健全性 ＋ CLAUDE.md/rules の剪定（削る側）
② ruleslesson-harvest
繰り返し指摘 → ルールへ追記（足す側）。「2回受けたら再発防止」を自動執行
③ skillsskill-audit
template 準拠・DRY/MECE・sprawl 閾値（8–12個）・未使用 disable
④ memorymemory-cleanup
cross-project 重複排除・L1 昇格候補・stale path 検出
⑤ staterecurring JSON
定期タスクを単一レジストリに集約。SessionStart hook が通知
03  ガバナンス — ローカル deterministic hook

危険操作はモデルに判断させず hook で止める

「責任領域（本番デプロイ承認・署名・対外発信）は AI に最終判断させない」を PreToolUse hook（governance-gate.py・632 行）で執行。コマンド文字列を解析し3挙動に振り分ける。

BLOCK 時は stderr に理由を返してモデルに再考させる。override は本人の手動実行のみ
04  記憶 — 3層モデル

決定論から semantic まで振り分ける

保存前に「2 project 以上で要る？ → L1／単一 PJ → L2／想起だけ → 放置（脳が自動 ingest）」を通す。cross-project に効く規約を単一 PJ に埋もれさせない。

確実に効かせたいものは L1、想起補助でよいものは L3 へ
grep
正規表現 / 完全一致 — 確定検索は L3 に投げない
query ~3s
semantic な「どこで？」 — 過去の決定を即引き出す
think ~57s
経緯の再構成 — 引用付きでまとめ直す
nightly
放っておくと育つ — dream で自己増殖

09  ローカル専用スタック
